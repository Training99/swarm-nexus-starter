# 责任闭环SOP模板
## Responsibility Closed-Loop SOP Template

**使用说明**: 将此模板导入你的OpenClaw系统，或作为手动执行检查清单。所有[方括号]内容需要替换为你的实际配置。

---

## 一、核心理念

### 1.1 什么是责任闭环？

**不是**: 任务分配下去就结束了  
**而是**: 确保任务被执行、有结果、有反馈、有优化

**关键区别**:
- ❌ 过程负责: "我做了我该做的"
- ✅ 结果负责: "我确保事情完成了"

### 1.2 责任闭环的四个环节

```
任务生成 → 指定责任人 → 执行落实 → 结果反馈 → 复盘优化
    ↑___________________________________________|
                    闭环完成
```

**环节说明**:
1. **任务生成**: 识别需求，明确目标
2. **指定责任人**: 必须是一个具体的人/Agent
3. **执行落实**: 责任人完成任务
4. **结果反馈**: 汇报完成情况
5. **复盘优化**: 总结经验，改进流程

## 二、每日执行流程

### 2.1 早晨启动 (建议时间: [08:00])

**执行人**: [你的执行总监Agent，如Athena]

**检查清单**:
- [ ] 检查昨日建议的执行状态
- [ ] 确认未完成任务的责任人
- [ ] 标记超时任务，准备升级

**输出**: 每日执行状态报告

**报告模板**:
```
【执行状态报告】[日期]

昨日建议总数: [X]条
已执行完成: [Y]条 ([Y/X]%)
执行中: [Z]条
超时未响应: [W]条 ⚠️

超时任务详情:
1. [任务描述] - 责任人:[Agent名] - 超时:[时长]
   建议: [升级/催促/取消]

今日待跟进:
1. [任务描述] - 责任人:[Agent名] - 截止时间:[时间]
```

---

### 2.2 任务生成与分配 (建议时间: [09:00])

**执行人**: [你的战略顾问Agent，如Sage]

**规则**: 每一条建议必须包含

| 要素 | 说明 | 示例 |
|-----|------|------|
| **任务描述** | 具体要做什么 | "分析昨日数据并生成报告" |
| **责任人** | 具体的Agent名字 | "@Alex" |
| **截止时间** | 何时必须完成 | "今日14:00前" |
| **交付标准** | 做到什么程度算完成 | "报告包含3个洞察+1个建议" |
| **升级条件** | 什么情况下通知你 | "超时2小时/遇到阻塞" |

**禁止的表述**:
- ❌ "大家看一下这个数据"
- ❌ "建议优化一下流程"
- ❌ "有时间处理一下"

**正确的表述**:
- ✅ "@Alex 分析昨日EvoMap数据，今日14:00前输出报告，需包含3个洞察。超时2小时未交付请通知我。"

---

### 2.3 任务执行 (全天)

**执行人**: 各Agent责任人

**责任人义务**:
1. **确认接收**: 收到任务后回复"收到，预计[时间]完成"
2. **进度同步**: 如遇阻塞，立即报告，不要沉默
3. **按时交付**: 在截止时间前交付成果
4. **质量自检**: 对照交付标准自检

**交付时必须包含**:
- ✅ 完成结果
- ✅ 关键数据/截图（如有）
- ✅ 下一步建议（如有）
- ✅ 遇到的问题（如有）

---

### 2.4 结果验收 (建议时间: [18:00])

**执行人**: [你自己] 或 [指定的验收Agent]

**验收清单**:
- [ ] 任务是否按时完成？
- [ ] 交付物是否符合质量标准？
- [ ] 是否达到预期目标？
- [ ] 是否需要优化迭代？

**验收结果**:
- 🟢 通过: 任务完成，进入复盘
- 🟡 需修改: 明确修改要求，重新进入执行
- 🔴 未通过: 分析原因，重新分配或取消

---

### 2.5 每日复盘 (建议时间: [23:00])

**执行人**: [你的复盘Agent，如Sage]

**复盘模板**:

```
【每日复盘】[日期]

```

## 三、质量标准清单

### 3.1 交付物质量标准

每个交付物必须通过以下检查：

**内容类交付物** (报告/分析/文案):
- [ ] 有明确的结论或产出
- [ ] 数据准确，有来源标注
- [ ] 逻辑清晰，结构完整
- [ ] 无错别字/语病
- [ ] 符合预期目标

**技术类交付物** (代码/配置/工具):
- [ ] 可正常运行/使用
- [ ] 有使用说明
- [ ] 已做基本测试
- [ ] 无安全隐患
- [ ] 符合技术规范

**创意类交付物** (设计/文案/策划):
- [ ] 符合品牌调性
- [ ] 有创意亮点
- [ ] 可执行落地
- [ ] 无版权风险
- [ ] 符合目标受众

### 3.2 "水下念诗禁令"示例

**不合格交付物的特征**:
- ❌ 只有过程描述，没有结果产出
- ❌ 质量明显低于可接受标准
- ❌ 需要别人大量返工才能使用
- ❌ 延迟交付且没有提前沟通

**遇到不合格交付物**:
1. 明确告知问题
2. 要求重新交付
3. 记录到复盘
4. 必要时调整责任人

## 四、升级机制

### 4.1 自动升级条件

以下情况自动升级通知你：

| 条件 | 升级时限 | 通知方式 |
|-----|---------|---------|
| 任务超时 | 超过截止时间2小时 | @你 + 发送消息 |
| 执行阻塞 | Agent报告无法继续 | 立即通知 |
| 质量不达标 | 验收不通过 | @你 + 说明原因 |
| 争议无法解决 | 多Agent扯皮 | @你 + 背景说明 |

### 4.2 升级模板

```
【任务升级通知】⚠️

任务: [任务描述]
责任人: [Agent名]
原定截止时间: [时间]
当前状态: [超时/阻塞/质量不达标]

问题描述:
[具体情况]

建议处理方案:
1. [方案A]
2. [方案B]

请确认处理方式。
```

## 六、导入配置

### 6.1 OpenClaw导入步骤

1. 将本SOP保存为 `RESPONSIBILITY_SOP.md`
2. 在OpenClaw中创建新Skill: "责任闭环管理"
3. 将本SOP内容复制到Skill的System Prompt中
4. 指定执行Agent（如Athena）
5. 设置定时触发（每日08:00、23:00）

### 6.2 手动执行

如果不使用OpenClaw，可将此SOP作为检查清单：
- 打印出来，放在桌面上
- 每日对照执行
- 用Excel记录任务状态

## 七、常见问题

**Q: 任务太多，每个都要明确责任人吗？**
A: 是的。如果任务小到不需要责任人，那它可能不值得做。或者，批量任务可以指定一个"批次责任人"。

**Q: Agent不配合，不回复怎么办？**
A: 检查你的Prompt是否明确。Agent不是人，它们需要非常清晰的指令。如果还是不配合，检查API Key和工具配置。

**Q: 质量标准怎么定？**
A: 从你的不满意案例中反推。把"水下念诗"作为反面教材，列出你不满意的点，就是质量标准。

**Q: 我每天要花多少时间管理这个系统？**
A: 初期每天30分钟（早15分钟+晚15分钟）。熟练后每天10分钟。系统帮你省下的时间，远不止这点投入。
